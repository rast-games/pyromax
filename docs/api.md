# API Reference

## MaxApi

Главный клиент библиотеки.

### Что умеет
- Инициализировать транспорт, протокол и маппер.
- Отправлять сообщения.
- Слушать обновления.
- Загружать файлы.
- Получать участников чата.

### Основные методы
- `send_message()`
- `download_file()`
- `get_member_by_id()`

## Dispatcher

Диспетчер обновлений и событий.

### Что делает
- Принимает апдейты от `MaxApi`.
- Прогоняет их через роутеры.
- Вызывает подходящие хендлеры.

### Основной метод
- `start_polling()`

## Router

Контейнер для хендлеров и дочерних роутеров.

### Что умеет
- Регистрировать события.
- Подключать другие роутеры.
- Передавать апдейты в нужные обработчики.

### Основные методы
- `include_router()`
- `include_routers()`
- `notify()`

## Message

Объект сообщения.

### Полезные поля и методы
- `text`
- `attaches`
- `reply()`
- `answer()`

## Filter

Базовый класс фильтров.

### Зачем нужен
Позволяет проверять входящие обновления и добавлять данные в контекст хендлера.

## CommandObject

Объект, который хранит распознанную команду.

### Поля
- `prefix`
- `command`
- `mention`
- `args`
